
## Options

```mdx
Usage: kubb [options]

Options:
  -c, --config <path>    Path to the Kubb config
  -i, --input <path>     Path of the input file(overrides the one in `kubb.config.js`)
  -l, --logLevel <type>  Type of the logging(overrides the one in `kubb.config.js`) (choices: "info", "silent", "stacktrace")
  --init                 Init Kubb
  -d, --debug            Debug mode (default: false)
  -w, --watch            Watch mode based on the input file
  -v                     Output the version number
  -h, --help             Display help for command
```

### --config (-c)

Path to the Kubb config.

```sh copy
npx kubb -c kubb.config.ts
```

### --input (-i)

Path of the input file(overrides the one in `kubb.config.js`)

```sh copy
npx kubb -i petStore.yaml
```

### --init

Init Kubb(`kubb.config.js` and install packages)

```sh copy
npx kubb --init
```

### --debug (-d)

Debug mode (default: false).

```sh copy
npx kubb -d
```

### --logLevel (-l)

Type of the logging(overrides the one in `kubb.config.js`)

silent` will hide all information that is not relevant

`info` will show all information possible(not related to the PluginManager)

`stacktrace` will show all information possible(related to the PluginManager), handy for seeing logs
 

```sh copy
npx kubb -d
```

### --watch (-w)

Watch mode based on the input file.

```sh copy
npx kubb -w
```

### --version (-v)

Output the version number

```sh copy
npx kubb -v
```

### --help (-h)

Display help for command

```sh copy
npx kubb -h
```
